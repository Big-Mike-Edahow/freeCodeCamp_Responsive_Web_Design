Introduction

The Go programming language was developed at Google by in 2007 by Robert Griesemer, Rob Pike, and Ken Thompson. It was announced to the public in November of 2009. Version 1.0 was released in March of 2012.

Go is a statically typed and compiled high level language that aims for simplicity. To quote Rob Pike: "Go's purpose is therefore not to do research into programming language design; it is to improve the working environment for its designers and their coworkers."

  • One of Go's key aspects is it's speed as a compiled language. It's standard library is optimized for performance and low resource usage.
  • Go's standard library contains the neccessary tools out of the box to build http servers and communicate with database management systems. Go has support for concurrency with goroutines and channels, garbage collection and cloud development.

Hello world

The traditional first program written when learning a new programming language is the quintessential "hello world", the origins of which can be traced back to the early 1970's. Brian Kernighan (the author of "The C Programming Language"), wrote the first version as part of a tutorial for the B programming language while working at Bell Labs.

package main import "fmt" func main() { fmt.Println("Hello world!") }
Variable declaration

Go allows you to declare a variable the following ways:

Using the var keyword. An example: var name string This declares a variable named name, of type string.

A variable can be declared and initialized by assigning a value to it. Code example: var num int = 25 This declares a variable named num, of type integer, and initializes it to the value of 25.

With the special := operator. The compiler will infer the variable's data type by the value assigned to it. Example: x := 13 This declares a variable named x with an initial value of 13.

Function declarations

User defined functions in Go are declared using the func keyword, the function name, parentheses, and curly brackets. This is also called the function signature. An example function follows: func squareRoot(num int) { num = num * num return num }

  • The name of the function cannot be a Go keyword.
  • Function names should be short and concise. They should clearly reflect the role that the function plays in the program.
  • Functions can be called in a program by using the function name followed by parentheses. After the function is executed, program flow will return to the point directly after the function call.
